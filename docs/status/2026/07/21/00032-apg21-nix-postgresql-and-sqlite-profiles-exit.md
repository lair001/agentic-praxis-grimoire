# APG21 Nix, PostgreSQL, and SQLite Profiles Exit

Phase ID: `APG21`

## Disposition

Partial — PostgreSQL and SQLite profiles implemented; Nix deferred

## Scope and outcome

APG21 accepts ADR 0016 and retains independent provisional PostgreSQL and
SQLite profiles after current-source research, separate threshold and semantic
analyses, private-guidance synthesis, shared design criticism, frozen
scenarios, read-only dogfood, focused tests, and fresh non-author acceptance.
No generic SQL or database-operations skill is adopted.

PostgreSQL uses one bounded measurement correction to define the exact
routine-body span and parser-unavailable decision behavior without changing a
threshold value. Nix uses one bounded measurement
and source-identity correction, then exposes a second behavior-bearing dynamic-
merge scenario contradiction and is `deferred-material-defect`. SQLite uses one bounded correction: the
`>= 13` rebuild-step Red threshold remains literal, and a
cohesive canonical rebuild may remain Orange only through the ordinary accepted
bounded-exception contract. No retained candidate has an unresolved material
defect; Nix evidence remains as the next fix-phase input.

## Profile results

The evaluated Nix candidate freezes fourteen concrete structural signals and separates
static parsing from evaluation, fetching, build, store, builder, activation,
and deletion authority. Its Red stops cover protected store exposure,
untrusted shell interpretation, mutable sources, material impurity, unsafe IFD,
invariant bypass, string-context removal, remote trust, activation, and
destructive store operations. Its closed dynamic-merge response remains
internally unresolved after the allowed correction, so no Nix leaf is retained.

The PostgreSQL profile freezes thirteen concrete structural signals while
keeping live relation size, locks, transaction age, WAL, replication,
privileges, and restore facts project-owned. Its Red stops cover injection,
unresolved live lock/rewrite risk, unbounded backfills and transactions, RLS
and definer-path bypass, false semantic labels, partial migrations, untested
backup reliance, protected data, extensions, replication, and unauthorized
live access.

The SQLite profile freezes twelve concrete structural signals while keeping
writer contention, journal state, filesystem behavior, permissions, and
restore facts project-owned. Its Red stops cover injection and extensions,
writer/retry assumptions, WAL/network and sidecar hazards, foreign-key and
integrity disablement, unsafe rebuilds, durability pragmas, row identity,
URI/path/`ATTACH` access, untested backup reliance, protected data, destructive
file replacement, and unauthorized live access.

All fifteen shared scenarios and 20 of 21 Nix scenarios pass; the failing
dynamic-merge scenario controls deferral. All shared and all 25 PostgreSQL and
25 SQLite scenarios pass. Dogfood is read-only and uses public maintained source,
official documentation examples, and public-safe synthetic operational cases.
No Nix evaluation, build, or activation and no database open or execution
occurred.
