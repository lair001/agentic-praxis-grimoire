## Contract and calibration

The accepted levels are Green routine, Yellow caution, Orange warning, and Red
crisis or stop. Levels classify one coherent current decision, do not grant
authority, and cannot relax superior safety, privacy, compatibility,
destructive-action, or task-authority rules. Orange requires an explicit local
decision, rationale, rollback, and focused validation. Red stops unsafe action
or new crisis-level growth until the governing condition is resolved.

Python defaults cover module lines, callable statements, cyclomatic complexity,
branches, nesting depth, explicit arguments, local bindings, and independent
responsibility families. They distinguish maintained hand-written code from
generated, vendored, protocol, migration, snapshot, machine-produced, test,
and legacy artifacts. Repository analyzers and stricter project policy remain
controlling.

Current Python, packaging, compatibility, typing, lint, and complexity sources
were inspected with their reuse terms. Numeric tool defaults and read-only
repository distributions informed the calibration; APG independently authored
the profile and added no tool dependency.
