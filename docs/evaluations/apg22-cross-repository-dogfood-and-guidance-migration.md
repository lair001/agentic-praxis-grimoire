## Guidance migration disposition

APG's root is already concise and owns authority, source treatment, retired-
workflow prohibition, universal work rules, and a project map. No APG root
reduction is justified.

RepoMap's root combines essential project policy with detailed phase, test,
Python, Go, PostgreSQL, and active-refactor guidance. A future RepoMap-owned
phase may shadow concise routes to existing contributor owners and public APG
profiles while retaining exact commands, thresholds, graph semantics, schema,
compatibility, runtime safety, and destructive-action policy in RepoMap. That
proposal requires current-source refresh, public v0.3 distribution, managed
projection, discovery and non-trigger evidence, override proof, target
acceptance, and byte-restorable rollback. It does not block APG publication.

Private global guidance remains the owner of personal, machine, integration,
host-escalation, and unsupported-language policy. Selected generic process and
profile judgment may shadow public APG only after v0.3 distribution and active
integration. The current private same-name router is an older transitional
six-process router, not semantic parity for the development public router. It
must remain until public distribution, active cutover, duplicate-name/source
handling, explicit use, non-trigger, override, and rollback evidence all pass.
The transition must establish source-qualified fresh-session discovery during
an explicit dual-source shadow before decommission and repeat it after cutover.
