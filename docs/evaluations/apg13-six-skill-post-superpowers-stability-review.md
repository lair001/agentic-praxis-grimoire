# APG13 Six-Skill Post-Superpowers Stability Review

## Objective and authority

APG13 accepts APG12A and reviews each current APG skill against the maturity
policy in ADR 0006 and the accepted criteria in ADR 0010. The phase may correct
one demonstrated material procedure defect per leaf, promote passing catalog
rows, validate the integrated development state, and update private development
history. It does not publish v0.2.0, mutate public/reference/RepoMap or active
integration, restore Superpowers, or begin APG14.

## Stable criteria and evidence classes

The review requires precise ownership and triggers, preserved authority,
coherent proportional procedure, repeated real use, representative
non-triggers, edge or stop evidence, safety and privacy, post-Superpowers
suitability, supported maintenance and rollback, current mechanical regression,
and fresh non-author review.

Positive real use, textual scenario application, non-trigger evidence,
integration evidence, mechanical evidence, distribution evidence, and
historical correction remain separate. Discovery, projection, installation,
release, catalog, and checker success do not become positive semantic use.

## Corrections, safety, and rollback

Every current leaf remains byte-identical to the APG12A baseline; APG13's
correction count is zero for all six. No unresolved authority, privacy, safety,
trigger, procedure, project-policy, evidence, stop, destructive-action, or
rollback defect remains. Maturity rollback restores the affected catalog label,
supersedes unsupported current statements, and reruns affected gates without
erasing historical records. Procedure rollback remains owned by each skill's
recorded lifecycle and correction history.

## Regression and mechanical evidence

The baseline and final integrated states passed the complete checker, release,
user, report-tool, and project-skill suites: 15 checker unit and 58 checker
integration tests; 8 release unit and 32 release integration tests; 8 user unit
and 39 user integration tests; 22 Bats report-tool families; and 28
project-skill integration families. Development and public v0.1.0 checker text
and JSON passed. Python compilation, Bash syntax/help, Markdown, local-link,
public/private, confidentiality, ADR, exit, mode, schema, projection, and Git
whitespace gates passed.

## Limitations

APG13 does not establish automatic invocation, clean baseline-versus-treatment
superiority, universal applicability, production warranty, a fixed test or task
count, or second-repository positive use for every leaf. Public v0.1.0 remains
unchanged and retains historical provisional metadata. Distribution and active
installation remain separate from current development maturity.

## APG14 boundary

All six current skills are `stable`, so the v0.2 maturity precondition for
APG14 is met. APG14 remains separately authorized and has not started. No real
v0.2.0 candidate, public tag, public push, publication, or active integration
update occurred, and this result grants no automatic successor authority.
