## APG23 closure

The required full application restart and new APG-rooted session occurred.
All nineteen repository skills were directly visible and selectable. The
repository workflow router was selected using client-supplied path evidence
that distinguished the same-name personal router. All thirteen provisional
skills passed individual explicit-use cases or retained a truthful stop, all
five required process/domain pairings passed, and no material defect remained.

Eight rows are promoted to stable and five remain provisional. Every v0.3
skill is `include-v0.3`. Provisional inclusion does not claim stable maturity;
stable promotion does not claim automatic invocation, comparative superiority,
universal applicability, production warranty, or operational authority.

The development state is 19 canonical leaves, 19 catalog rows, 19 checked-in
projections, 14 stable rows, 5 provisional rows, and 18 non-router routes.

## APG24 release closure

APG24 is the authorized and terminal v0.3 release phase. ADR 0019 distributes
all nineteen included skills, retains fourteen stable and five provisional
rows, makes user transitions source-release-specific, and expands new project
defaults while preserving existing subsets. Public v0.3.0 appends one squashed
commit and one annotated tag over unchanged v0.2.0. The active source advances
by fast-forward without changing aggregate-link ownership.

Root cutover, private-skill decommission, and personal-router replacement do
not occur. The personal and public routers remain together for a
source-qualified fresh-session shadow. Reference, target, RepoMap, database,
Nix, graph, and deployment state remain unchanged. No successor phase is
authorized.
